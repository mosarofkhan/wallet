React Modules Setup Guide

This guide will show you how to use @rainbow-me/rainbowkit, wagmi, viem, and @tanstack/react-query modules in a new React app using the zip files you downloaded from GitHub.

Step 1: Download the Zip

Download your zip file from GitHub:

https://github.com/mosarofkhan/wallet.git

Unzip it. Make sure the folder is named node_modules.

Step 2: Create a New React App

npx create-react-app my-new-app
cd my-new-app

Step 3: Replace node_modules

Copy the node_modules folder from your unzipped file into the new React app folder.

Also copy package.json and package-lock.json from the zip.

Step 4: Test the App

npm start

Your modules should work exactly like they would if installed from npm registry.

Note: If you need to add new dependencies or update versions, npm will download them from the registry. If the npm registry becomes unavailable, you can still work using the zip copy of node_modules.

Step 5: Using the Modules

Example code:

import React from 'react';
import { RainbowKitProvider } from '@rainbow-me/rainbowkit';
import { WagmiConfig, createClient } from 'wagmi';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { createPublicClient, http } from 'viem';

const client = createClient({
    autoConnect: true,
    connectors: [],
});

const queryClient = new QueryClient();

function App() {
    return (
        <WagmiConfig client={client}>
            <RainbowKitProvider chains={[]}>
                <QueryClientProvider client={queryClient}>
                    <div>Hello World</div>
                </QueryClientProvider>
            </RainbowKitProvider>
        </WagmiConfig>
    );
}

export default App;

Final Notes

This is how you can easily use your zip backup of modules in a new React project without relying on npm registry availability.
